## 1. Introduction

### 1.1 Document Purpose

Tài liệu này đặc tả đầy đủ các yêu cầu của hệ thống Blog cá nhân về lập trình — định nghĩa hệ thống phải làm gì, không mô tả cách triển khai. Phục vụ cho: tác giả/product owner (xác nhận scope), engineering (hiểu behavior), QA (thiết kế test cases). Tài liệu nguồn: PRD v1.0 (`docs/020_prd.md`). Chi tiết kiến trúc và implementation nằm trong tài liệu kỹ thuật riêng.

### 1.2 Product Scope

**Blog cá nhân về lập trình** — hệ thống web tĩnh cho phép tác giả chia sẻ kiến thức lập trình bằng cách quản lý nội dung trên Notion; bài viết tự xuất hiện trên web sau khi công bố, không cần thao tác thủ công. Người đọc có thể tìm kiếm, lọc theo tag, bình luận, theo dõi bài mới qua email/RSS, và đọc offline trên thiết bị di động.

Phạm vi bao gồm toàn bộ 8 phase theo roadmap trong PRD. Không bao gồm: công cụ soạn nội dung tự xây, multi-author, ứng dụng mobile native, đa ngôn ngữ.

## 2. Product Overview

### 2.1 Product Perspective

Hệ thống mới — không thay thế hệ thống hiện có. Blog hoạt động như một trang web tĩnh được gen từ nội dung lưu trên Notion. Luồng chính:

```
Notion (nguồn nội dung)
  → Sync pipeline (webhook + cron)
    → MDX files trong Git repository
      → Build & Deploy (Vercel)
        → Blog trên web (người đọc)
```

Các external systems phụ thuộc: Notion API (nguồn nội dung), GitHub API (lưu trữ nội dung sau sync), Vercel (hosting, cron, image optimization, KV, Blob), Giscus (bình luận), Resend (email thông báo), Vercel Analytics (lượt truy cập).

### 2.2 Product Functions

Các nhóm chức năng chính:

- **Đọc & khám phá nội dung:** Danh sách bài, tìm kiếm, lọc theo tag, sắp xếp theo ngày/phổ biến, trang chi tiết bài
- **SEO & theo dõi:** Sitemap, RSS feed, Open Graph metadata
- **Tương tác:** Đếm lượt xem, bình luận (Giscus), đăng ký nhận tin (Resend)
- **Quản lý nội dung:** Sync tự động từ Notion (webhook real-time + cron hằng ngày), xem trước bài draft
- **Trải nghiệm:** Dark/light mode, giao diện responsive, PWA (cài như app, đọc offline)
- **Bảo mật:** Xác thực webhook, draft preview với secret token, không lộ credential

### 2.3 Product Constraints

- **Nguồn nội dung:** Notion là CMS duy nhất — tác giả viết và công bố bài trên Notion. Constraint này là kết quả của quyết định trong PRD v1.0 (OQ-1 closed).
- **Hosting:** Vercel — cung cấp cron jobs, KV store, Blob storage, image optimization, deployment pipeline.
- **Build độc lập:** Build PHẢI hoàn thành không phụ thuộc khả dụng của Notion API tại thời điểm build (đây là ràng buộc cốt lõi tránh lỗi timeout đã xảy ra).
- **Framework:** Next.js App Router — ràng buộc kỹ thuật đã chốt.
- **Bình luận:** Giscus (GitHub Discussions) — PRD OQ-2 closed.
- **Email:** Resend — PRD OQ-4 closed.
- **Không có AI/ML component** — không áp dụng Section 3.6.
- **Geist Design System:** Giao diện theo phong cách Vercel Geist — tối giản, đen trắng chủ đạo, typography rõ ràng.

### 2.4 User Characteristics

| Persona | Vai trò | Tần suất | Trình độ kỹ thuật |
|---------|---------|---------|------------------|
| P-01 Tác giả | Quản lý nội dung trên Notion; xem trước bài draft; monitor sync | Định kỳ khi viết bài | Cao (developer) |

Accessibility: giao diện mobile-first, touch target đủ lớn (≥ 44×44px), không yêu cầu WCAG compliance chính thức (blog cá nhân).

### 3.1 External Interfaces

#### 3.1.1 User Interfaces

Giao diện web, hỗ trợ trình duyệt Chrome, Firefox, Safari phiên bản mới nhất. Thiết kế theo Geist Design System (2.3). Responsive mobile-first; các trang chính: trang chủ (danh sách bài), trang chi tiết bài, trang tag, kết quả tìm kiếm. Thông báo lỗi rõ ràng cho mọi trạng thái không tìm thấy / offline. Chi tiết layout → tài liệu UI/design riêng.

#### 3.1.2 Hardware Interfaces

N/A — phần mềm thuần web, không tương tác hardware trực tiếp.

#### 3.1.3 Software Interfaces

| Hệ thống | Vai trò | Giao thức / Ghi chú |
|---------|---------|-------------------|
| Notion API | Nguồn nội dung (fetch bài, blocks, ảnh) | HTTPS REST; đọc tại sync time, không tại build time |

| Vercel Blob | Lưu trữ ảnh từ Notion | HTTPS; URL stable, có CDN |
| Vercel Analytics | Số liệu lượt truy cập | Embed script |

#### REQ-FUNC-001: Danh sách bài viết đã công bố

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Trang chủ PHẢI hiển thị tất cả bài viết có trạng thái Published, sắp xếp theo ngày giảm dần theo mặc định, phân trang 10 bài mỗi trang.
- **Rationale:** Năng lực đọc cơ bản; US-001 PRD.
- **Acceptance Criteria:**
  - AC1 (happy path):
    - Given: có 15 bài Published
    - When: truy cập trang chủ (không có query param)
    - Then: hiển thị 10 bài mới nhất; có điều hướng đến trang 2
  - AC2 (negative):
    - Given: bài ở trạng thái Draft
    - When: tải trang chủ
    - Then: bài Draft không xuất hiện trong danh sách
  - AC3 (empty state):
    - Given: không có bài Published nào
    - When: tải trang chủ
    - Then: hiển thị thông báo rõ ràng (không phải danh sách trống im lặng)
- **Verification Method:** Test
- **More Information:** Traceability: US-001. Phase 1.

#### REQ-FUNC-002: Sắp xếp danh sách bài

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Hệ thống PHẢI cho phép người đọc chuyển đổi thứ tự sắp xếp danh sách bài sang "phổ biến nhất" (theo lượt xem giảm dần); nếu giá trị tham số sắp xếp không hợp lệ, hệ thống PHẢI tự dùng lại mặc định (ngày giảm dần) và không hiển thị lỗi.
- **Rationale:** US-002 PRD.
- **Acceptance Criteria:**
  - AC1:
    - Given: có nhiều bài với lượt xem khác nhau
    - When: truy cập `/?sort=popular`
    - Then: bài có lượt xem cao nhất hiển thị trước
  - AC2:
    - Given: URL có `?sort=invalid_value`
    - When: tải trang
    - Then: danh sách sắp xếp theo ngày giảm dần; không có thông báo lỗi
- **Verification Method:** Test
- **More Information:** Traceability: US-002. Phase 2.

#### REQ-FUNC-003: Lọc bài theo tag qua URL

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Hệ thống PHẢI cho phép lọc danh sách bài theo tag; trạng thái lọc PHẢI phản ánh trên URL; điều hướng back/forward của trình duyệt PHẢI hoạt động đúng.
- **Rationale:** US-003 PRD.
- **Acceptance Criteria:**
  - AC1:
    - Given: có bài với tag "nextjs" và tag "react"
    - When: chọn lọc tag "nextjs"
    - Then: URL cập nhật thành `/?tag=nextjs`; chỉ bài có tag "nextjs" được hiển thị
  - AC2:
    - Given: đang xem `/?tag=nextjs`
    - When: nhấn nút Back trình duyệt
    - Then: quay về trạng thái trước khi lọc đúng với URL trước đó
  - AC3:
    - Given: URL `/?tag=nextjs` được chia sẻ
    - When: mở link này
    - Then: trang hiển thị đúng danh sách bài lọc theo tag "nextjs"
- **Verification Method:** Test
- **More Information:** Traceability: US-003. Phase 2.

#### REQ-FUNC-004: Trang danh sách bài theo tag

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Mỗi tag PHẢI có trang riêng liệt kê bài thuộc tag đó; nếu tag không có bài nào, hệ thống PHẢI hiển thị thông báo rõ ràng; nếu tag không tồn tại, hệ thống PHẢI trả về 404.
- **Rationale:** US-004 PRD.
- **Acceptance Criteria:**
  - AC1:
    - Given: tag "nextjs" có 3 bài Published
    - When: truy cập `/tags/nextjs`
    - Then: hiển thị đúng 3 bài đó
  - AC2:
    - Given: tag "nextjs" tồn tại nhưng không có bài nào
    - When: truy cập `/tags/nextjs`
    - Then: hiển thị thông báo "Chưa có bài viết nào cho tag này" hoặc tương đương
  - AC3:
    - Given: tag "nonexistent" không tồn tại
    - When: truy cập `/tags/nonexistent`
    - Then: trả về HTTP 404
- **Verification Method:** Test
- **More Information:** Traceability: US-004. Phase 2.

#### REQ-FUNC-005: Tìm kiếm bài viết phía client

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Hệ thống PHẢI cung cấp tìm kiếm bài viết theo tiêu đề và nội dung; kết quả PHẢI hiển thị ngay khi nhập từ khóa, không có độ trễ chờ server; chỉ bài Published được trả về.

    - Given: bài ở trạng thái Draft có chứa từ khóa

- **More Information:** Traceability: US-005. Phase 2. Constraint: search index xây dựng tại build time.

#### REQ-FUNC-006: Trang chi tiết bài viết

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Trang chi tiết PHẢI hiển thị đầy đủ nội dung bài viết bao gồm: heading các cấp, đoạn văn, danh sách, đoạn code có syntax highlighting, ảnh; nếu bài không tồn tại hoặc chưa Published, hệ thống PHẢI trả về 404 (trừ khi truy cập qua draft preview — xem REQ-FUNC-017).
- **Rationale:** US-006 PRD.
- **Acceptance Criteria:**
  - AC1:
    - Given: bài Published với đủ các loại nội dung
    - When: truy cập `/posts/{slug}`

  - AC2:
    - Given: slug không tồn tại
    - When: truy cập `/posts/nonexistent-slug`
    - Then: trả về HTTP 404
  - AC3:
    - Given: bài ở trạng thái Draft
    - When: truy cập `/posts/{slug}` bình thường (không qua draft link)
    - Then: trả về HTTP 404
- **Verification Method:** Test + Demonstration
- **More Information:** Traceability: US-006. Phase 1.

##### REQ-SEC-001: Xác thực webhook signature

- **Status:** draft
- **Priority:** High
- **Owner:** @author
- **Statement:** Endpoint nhận Notion webhook PHẢI xác thực signature trong header của mọi request; request không có signature hoặc signature không hợp lệ PHẢI bị từ chối với HTTP 401 và KHÔNG gây ra bất kỳ thay đổi nào.
- **Rationale:** PRD Section 5 Security; ngăn giả mạo trigger sync.
- **Acceptance Criteria:**
  - AC1:
    - Given: request đến webhook endpoint không có signature header
    - When: handler xử lý
    - Then: trả về HTTP 401; không có sync nào được thực thi; log ghi nhận request bị từ chối
  - AC2:
    - Given: request có signature sai
    - When: handler xử lý
    - Then: trả về HTTP 401; không có thay đổi nào được commit
  - AC3:
    - Given: request hợp lệ với signature đúng
    - When: handler xử lý
    - Then: sync được thực thi bình thường
- **Verification Method:** Test
- **More Information:** Verification challenge (≤3s) cũng phải được xử lý trước khi verify signature. Cross-ref REQ-SEC-003.
